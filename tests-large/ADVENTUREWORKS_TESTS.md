## Bugs Fixed During Test Development

1. **`renderJoin` schema qualification** (`core/internal/psql/query.go:614`): Intermediate JOIN tables were rendered without schema prefix (e.g., `INNER JOIN employee` instead of `INNER JOIN humanresources.employee`).
2. **Composite FK columns missing from subquery SELECT** (`core/internal/qcode/fields.go:247`): Extra pair columns in composite FKs were not added to the parent subquery's base columns, causing "column does not exist" errors on aliased subqueries.
3. **FK column disambiguation in WHERE** (`core/internal/qcode/exp.go`): FK columns like `customer_id` were misinterpreted as relationship joins instead of simple column filters.

## Features Exercised

- **Cross-schema joins**: Tests 10, 14, 15, 18, 19, 20, 24 join across `sales`, `person`, `production`, `humanresources`, `purchasing` schemas
- **Composite foreign keys**: Tests 9, 10 verify multi-column FK joins `(productid, specialofferid)`
- **Deep nesting**: Tests 10 (5 levels), 20 (7 tables), 24 (6 levels)
- **Fan-out queries**: Tests 5, 6, 16, 19 join multiple children from one parent
- **Many-to-many**: Test 22 traverses junction table
- **Large dataset**: Test 13 queries 121K rows with ordering
- **Filter predicates**: Tests 11, 17 use `is_null`, Tests 4, 13 use `gt`/`desc`
- **Aggregation order_by**: Test 25 orders by `sum_orderqty` (aggregation function in order_by)
- **Variable binding**: Test 17 uses `$name` variable, Test 25 uses `$tid`
- **Schema discovery**: Test 2 validates 160-table cross-schema discovery
- **Ground truth verification**: Every test runs the equivalent SQL query first and compares results
